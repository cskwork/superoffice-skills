# examples - 형식 따라하기 (하우스 견본 + 사용자 예시)

Load from `reference/office.md` when 사용자가 "이 형식으로 / 이거 따라서 만들어줘" 하거나, 일관된 사내 포맷이 필요할 때. 추상적 지시 대신 **구체 예시의 구조를 모방**해 형식을 맞춘다.

## 두 가지 예시 소스

1. **하우스 견본** (`examples/format/`): `status-report.md`(진행 현황 7섹션), `proposal.md`(제안 SCQA), `exec-summary.md`(1페이지 요약). biz-report 구조를 자리표시로 채운 형식 모델 - 형식 무관(어느 산출 형식으로도 렌더). 사용자가 형식을 따로 안 줬는데 표준 보고가 필요하면 여기서 고른다.
2. **사용자 제공 예시** (또는 회사 양식 파일): 사용자가 준 기존 문서(.docx/.pptx/.xlsx/.hwpx/.pdf)나 회사 양식. `reference/doc-ingest.md`로 읽어 구조를 뽑는다. 회사 양식 파일은 브랜드(색/폰트)와 형식(레이아웃) **이중 소스**다 - 브랜드는 `reference/brand-kit.md`, 형식은 여기.

## 절차 - 구조를 따르되 내용은 새로 쓴다

1. **예시 읽기**: 하우스 견본은 그대로, 사용자 예시는 `doc-ingest`로 본문/표/제목을 추출.
2. **형식 스켈레톤 추출** (내용 아님): 섹션 순서·제목 스타일·표 모양(열 구성)·톤(존댓말/문체)·길이(한장/멀티)·번호 체계. "무엇을 말하는가"가 아니라 "어떻게 배열·표기하는가"를 뽑는다.
3. **새 내용으로 채움**: 이번 브리프의 실제 내용을 그 스켈레톤에 넣는다. 예시의 문장·수치를 **verbatim 복제하지 않는다**(저작권·위조). 예시는 틀이지 내용이 아니다.
4. **biz-report 규율은 예시보다 우선**: 예시 제목이 주제 라벨이면 action-title(결론)로 올리고, RAG·So-what·MECE를 적용한다. 예시가 약한 부분은 따라 내려가지 말고 `reference/biz-report.md` 기준으로 보강한다.
5. **브랜드 결합**: 회사 양식이 소스면 형식(여기) + 색/폰트(`brand-kit.json`)를 함께 입힌다.

## 무결성 (위조 금지)

- 예시의 수치/사실을 그대로 옮기지 않는다 - 이번 산출의 수치는 `facts.json` 출처 또는 자리표시(`[NN]`/`[출처]`). 견본의 자리표시를 그대로 남기지 말고 실제 값/출처로 채우거나 명시적 placeholder로 둔다.
- 사용자 예시 문서의 저작권·기밀을 존중한다(사내 한정 표기 보존, 외부 유출 금지).
- 스캔 PDF 등 추출이 빈 예시는 "형식 추출 불가"로 명시하고 하우스 견본으로 강등한다(빈 구조를 추정으로 채우지 않는다).

## 게이트

형식을 따라 만든 산출물도 동일하게 `bash templates/office-gate.sh <vault> <text files>`. 견본 자체(`examples/format/*.md`)도 하우스 규칙(이모지 없음·어절 띄어쓰기·자리표시 수치)을 지킨다.
